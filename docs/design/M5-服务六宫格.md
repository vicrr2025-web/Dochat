## 一、功能范围

M5 是服务 Tab 的主页面，核心是**八大生态入口聚合 + 角标联动**。不包含任何生态的具体业务逻辑（M7-M14 各自独立实现），点击入口跳转到"开发中"占位页。

### 功能闭环

| # | 功能 | 描述 |
|:---:|------|------|
| 1 | 六宫格入口 | 3行×3列网格（第3行2个），展示八大生态图标+名称+角标数 |
| 2 | 角标数字显示 | 每个入口卡片右上角显示红色角标数（0 时隐藏） |
| 3 | 角标级联聚合 | 生态事件 → 入口角标 → 服务Tab角标 三级联动 |
| 4 | 最近使用 | 底部显示最近访问的生态入口（最多4个） |
| 5 | 点击入口 | 跳转"开发中"占位页，显示生态名称 + Coming Soon |
| 6 | 服务Tab角标 | 底部导航服务Tab上显示总角标数（所有入口角标之和） |
| 7 | 点击记录 | 点击入口 → 写入最近使用列表 + 触发角标更新 |
| 8 | 下拉刷新 | 拉取最新角标数据 |

## 二、数据模型

### 2.1 后端（1 张表）

#### `ecosystem_badges` — 角标计数表

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT PK AUTO | 主键 |
| `user_id` | BIGINT NOT NULL | 用户ID |
| `ecosystem_key` | VARCHAR(32) NOT NULL | 生态标识（escrow/mall/dating/housing/jobs/mail/logistics/home_service） |
| `badge_count` | INT DEFAULT 0 | 当前角标数 |
| `updated_at` | TIMESTAMP | 更新时间 |

- 唯一约束：`UNIQUE(user_id, ecosystem_key)`
- M5 阶段通过 Mock API 返回模拟数据（真实数据由 M7-M14 各生态推送）
- 预留 `POST /api/v1/services/badges/update` 供各生态调用

## 三、API 设计

### 3.1 后端 API（2 个）

| 方法 | 路径 | 说明 | 鉴权 |
|------|------|------|:---:|
| GET | `/api/v1/services/badges` | 获取当前用户全部生态角标数 | JWT |
| POST | `/api/v1/services/badges/update` | 更新指定生态角标数（增量，供 M7-M14 调用） | JWT |

#### GET `/api/v1/services/badges` 响应

```json
{
  "code": 0,
  "data": {
    "badges": [
      { "ecosystemKey": "escrow", "badgeCount": 3 },
      { "ecosystemKey": "mall", "badgeCount": 0 },
      { "ecosystemKey": "dating", "badgeCount": 5 },
      { "ecosystemKey": "housing", "badgeCount": 0 },
      { "ecosystemKey": "jobs", "badgeCount": 1 },
      { "ecosystemKey": "mail", "badgeCount": 12 },
      { "ecosystemKey": "logistics", "badgeCount": 0 },
      { "ecosystemKey": "homeService", "badgeCount": 2 }
    ],
    "totalBadge": 23
  }
}
```

#### POST `/api/v1/services/badges/update` — 增量更新角标

请求体：
```json
{
  "ecosystemKey": "mail",
  "delta": 1
}
```

- `delta`：正数=增加，负数=减少，0=重置
- 后端逻辑：`badgeCount += delta`，不小于 0

响应：
```json
{
  "code": 0,
  "data": {
    "ecosystemKey": "mail",
    "badgeCount": 13
  }
}
```

### 3.2 最近使用策略

**最近使用记录走纯前端本地存储（SharedPreferences），不走后端 API。**
- 点击入口 → `SharedPreferences` 写入 `recentKeys` 列表（去重，最新在前，最多 4 个）
- 页面加载时从本地读取，无需网络请求
- 不需要 `GET /api/v1/services/recent` 接口，不需要后端 `RecentResponse` DTO

### 3.3 前端 API 调用

前端 Service 层新增 `ServiceHubService`：

| 方法 | HTTP | 路径 |
|------|------|------|
| `fetchBadges()` | GET | `/v1/services/badges` |

## 八、Provider 状态管理

```dart
class ServiceHubProvider extends ChangeNotifier {
  List<EcosystemBadge> _badges = [];
  List<String> _recentKeys = [];
  bool _isLoading = false;

  List<EcosystemBadge> get badges => _badges;
  List<String> get recentKeys => _recentKeys;
  bool get isLoading => _isLoading;
  int get totalBadge => _badges.fold(0, (sum, b) => sum + b.badgeCount);

  Future<void> fetchBadges();
  Future<void> fetchRecent();
  void recordVisit(String ecosystemKey);  // 前端本地记录
}
```
