## 三、数据模型（修订）

### sessions 表
```sql
CREATE TABLE sessions (
    id            BIGSERIAL PRIMARY KEY,
    session_id    VARCHAR(32) UNIQUE NOT NULL,
    type          VARCHAR(20) NOT NULL DEFAULT 'single',   -- single
    name          VARCHAR(100),
    avatar        VARCHAR(500),
    last_message  TEXT,
    last_message_type VARCHAR(20),
    last_time     TIMESTAMP,
    is_pinned     BOOLEAN DEFAULT FALSE,
    is_muted      BOOLEAN DEFAULT FALSE,
    mute_expire   TIMESTAMP,
    created_at    TIMESTAMP DEFAULT NOW(),
    updated_at    TIMESTAMP DEFAULT NOW()
);
```

### session_members 表
```sql
CREATE TABLE session_members (
    id            BIGSERIAL PRIMARY KEY,
    session_id    VARCHAR(32) NOT NULL,
    user_id       VARCHAR(32) NOT NULL,
    unread_count  INTEGER DEFAULT 0,
    joined_at     TIMESTAMP DEFAULT NOW(),
    UNIQUE(session_id, user_id)
);
```

### messages 表
```sql
CREATE TABLE messages (
    id            BIGSERIAL PRIMARY KEY,
    message_id    VARCHAR(32) UNIQUE NOT NULL,
    session_id    VARCHAR(32) NOT NULL,
    sender_id     VARCHAR(32) NOT NULL,
    type          VARCHAR(20) NOT NULL,          -- text/voice/image/file/system
    content       TEXT,
    media_url     VARCHAR(500),
    media_duration INTEGER,                      -- 语音时长(秒)
    file_name     VARCHAR(200),
    file_size     BIGINT,
    status        VARCHAR(20) DEFAULT 'sent',    -- sent/delivered/read/revoked/failed
    is_recalled   BOOLEAN DEFAULT FALSE,
    read_at       TIMESTAMP,
    sent_at       TIMESTAMP DEFAULT NOW(),
    recalled_at   TIMESTAMP
);
CREATE INDEX idx_messages_session ON messages(session_id, sent_at DESC);
```

## 五、核心实现策略（补充细节）

### 5.1 Mock 消息收发流程

```
发送方:
  POST /api/v1/messages
    → 写入 messages 表 (status=sent, sent_at=now)
    → 更新 sessions.last_message / last_time
    → 返回 { messageId, status:"sent" }
    → 发布 Redis Pub/Sub: channel "chat:{sessionId}"

接收方（在线用户通过 SSE 实时接收）:
  GET /api/v1/sessions/{id}/subscribe (SSE端点)
    → 连接保持
    → Redis Pub/Sub consumer → 收到消息 → 推送 SSE event
    → 前端收到 → 更新 messages 表 (status=delivered)
    → 前端进入聊天页 → PUT /api/v1/messages/read (status=read)
```

### 5.2 SSE 长连接端点（补充）

```
GET /api/v1/sessions/{sessionId}/subscribe
  Header: Authorization: Bearer {token}
  Response: text/event-stream
    event: message
    data: { "messageId":"...", "type":"text", "content":"hello", ... }
    
    event: typing
    data: { "userId":"...", "typing":true }
    
    event: read
    data: { "userId":"...", "lastMessageId":"..." }
```

### 5.3 文件上传安全策略（Mock）

- 类型白名单：图片(png/jpg/gif/webp)、文件(pdf/doc/xlsx/txt/zip)
- 大小限制：图片≤5MB，文件≤50MB
- 存储路径：`uploads/{yyyy-MM}/{uuid}.{ext}`
- 防路径遍历：校验文件名不含 `../`

### 5.4 消息分页策略

- **会话列表**：偏移分页 `?page=0&size=20`
- **历史消息**：游标分页 `?sessionId=&before={messageId}&size=20`
  - 首次请求不传 `before`，返回最新 20 条
  - 加载更多传最后一条的 `messageId`
