## 二、数据模型（2 新表）

```sql
-- 担保交易表
CREATE TABLE guarantee_trades (
    id BIGSERIAL PRIMARY KEY,
    trade_id VARCHAR(32) NOT NULL UNIQUE,
    buyer_id VARCHAR(32) NOT NULL,
    seller_id VARCHAR(32) NOT NULL,
    product_name VARCHAR(256),
    product_desc TEXT,
    amount DECIMAL(10,2) NOT NULL,
    status VARCHAR(16) DEFAULT 'pending',  -- pending/confirmed/frozen/verifying/verified/releasing/completed/cancelled/disputed
    verify_status VARCHAR(16),             -- null/pending/passed/failed
    verify_report TEXT,
    session_id VARCHAR(32),                -- 内嵌聊天会话 ID
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 纠纷表
CREATE TABLE guarantee_disputes (
    id BIGSERIAL PRIMARY KEY,
    dispute_id VARCHAR(32) NOT NULL UNIQUE,
    trade_id VARCHAR(32) NOT NULL,
    initiator_id VARCHAR(32) NOT NULL,
    reason VARCHAR(64),
    description TEXT,
    status VARCHAR(16) DEFAULT 'pending',  -- pending/voting/verdict_executed
    verdict VARCHAR(16),                   -- buyer_win/seller_win
    jury_count INT DEFAULT 0,
    votes_for_buyer INT DEFAULT 0,
    votes_for_seller INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT NOW(),
    resolved_at TIMESTAMP
);
```

### 4.1 交易详情页状态流

```
pending ──(卖家确认)──→ confirmed ──(买家确认付款)──→ frozen
frozen ──(发起验号)──→ verifying ──(验号通过)──→ verified
verified ──(买家确认收货)──→ releasing ──(释放资金)──→ completed
任意状态 ──(发起纠纷)──→ disputed ──(裁决)──→ completed/cancelled
verified ──(验号不通过+退款)──→ cancelled
```
