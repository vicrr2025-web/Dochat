| 输入手机号（11位）→ 点击获取验证码 → 后端生成6位验证码 → 发送短信（Mock：直接返回验证码，开发阶段显示在Toast） → 输入验证码 → 设置密码（6-20位，含字母+数字）→ 确认密码 → 点击注册 → 后端校验 → 创建用户 → 生成JWT + UserSig → 自动登录 → 跳转主页 | 手机号格式错误 → 输入框下方红色提示"请输入正确的手机号" |

| | 验证码错误 → 输入框下方红色提示"验证码错误" |
| | 验证码过期（5分钟）→ Toast提示"验证码已过期，请重新获取" |

### 3.4 自动登录

| 正常逻辑 | 异常边界 |
|----------|----------|

| | UserSig过期 → 重新调用腾讯IM生成UserSig → 更新本地 → 跳转主页 |
| | 本地无Token → 跳转登录页 |
| | 网络异常 → 跳转登录页 |

### 4.2 接口详细定义

#### 4.2.1 发送短信验证码

```
POST /api/auth/sms/send

Request:
{
  "phone": "13800138000",       // 手机号，11位，必填
  "type": "register"            // 类型：register | login | reset，必填
}

Response 成功:
{
  "code": 0,
  "message": "验证码已发送",
  "data": {
    "expireIn": 300,             // 有效期（秒）
    "retryAfter": 60             // 重发间隔（秒）
  }
}

Response 失败:
{
  "code": 1001,                  // 错误码
  "message": "手机号格式错误"
}

字段校验规则：
- phone：必填，正则 ^1[3-9]\d{9}$，不符合返回1001
- type：必填，枚举 register|login|reset，不符合返回1002

Mock实现（开发阶段）：
- 不调用腾讯云短信API
- 生成6位随机验证码，存入Redis（key: sms:{phone}:{type}，过期300秒）
- 通过日志输出验证码（实际日志级别DEBUG）
- 同一手机号同一类型60秒内不可重复发送，返回1003
- 同一手机号每日上限10次，返回1004
```

#### 4.2.2 手机号注册

```
POST /api/auth/register

Request:
{
  "phone": "13800138000",       // 手机号，必填
  "smsCode": "123456",          // 验证码，必填，6位数字
  "password": "abc12345"        // 密码，必填，6-20位
}

Response 成功:
{
  "code": 0,
  "message": "注册成功",
  "data": {
    "userId": "u_xxx",
    "token": "eyJhbG...",        // JWT，有效期24h
    "refreshToken": "eyJhb...",  // RefreshToken，有效期30天
    "userSig": "eJwtz...",       // 腾讯IM UserSig
    "expiresIn": 86400           // Token过期时间（秒）
  }
}

字段校验规则：
- phone：同4.2.1
- smsCode：必填，6位数字，从Redis校验，过期返回1007
- password：必填，正则 ^(?=.*[a-zA-Z])(?=.*\d).{6,20}$，不符合返回1005

业务校验：
- 手机号已注册 → 返回1006 "该手机号已注册"
- Redis中验证码过期 → 返回1007 "验证码已过期"
- Redis中验证码不匹配 → 返回1008 "验证码错误"
```

#### 4.2.3 密码登录

```
POST /api/auth/login

Request:
{
  "phone": "13800138000",       // 手机号，必填
  "password": "abc12345"        // 密码，必填
}

Response 成功:
{
  "code": 0,
  "message": "登录成功",
  "data": {
    "userId": "u_xxx",
    "token": "eyJhbG...",
    "refreshToken": "eyJhb...",
    "userSig": "eJwtz...",
    "expiresIn": 86400
  }
}

Response 失败:
{
  "code": 1009,
  "message": "密码错误，还剩4次尝试机会"
}

字段校验：
- phone：同4.2.1
- password：必填，非空

业务校验：
- 手机号未注册 → 返回1010 "该手机号未注册"
- 密码错误 → 返回1009，Redis计数+1
- 密码错误5次 → 账号锁定30分钟 → 返回1011 "账号已锁定，请30分钟后重试"
- 账号被禁用 → 返回1012 "账号已被禁用，请联系客服"
```

#### 4.2.4 验证码登录

```
POST /api/auth/login/sms

Request:
{
  "phone": "13800138000",
  "smsCode": "123456"
}

Response 成功:
{
  "code": 0,
  "message": "登录成功",
  "data": {
    "userId": "u_xxx",
    "token": "eyJhbG...",
    "refreshToken": "eyJhb...",
    "userSig": "eJwtz...",
    "expiresIn": 86400
  }
}

业务逻辑：
- 手机号未注册 → 自动注册（生成随机密码 + 创建用户）→ 返回2001（首次注册标识）
- 手机号已注册 → 正常登录
- 验证码校验 → 同注册
```

#### 4.2.5 Token刷新

```
POST /api/auth/refresh

Request Header:
Authorization: Bearer {refreshToken}

Response 成功:
{
  "code": 0,
  "message": "Token刷新成功",
  "data": {
    "token": "eyJhbG...",
    "refreshToken": "eyJhb...",
    "expiresIn": 86400
  }
}

业务逻辑：
- RefreshToken从Redis校验
- 过期 → 返回1013 "Token已过期，请重新登录"
- 无效 → 返回1014 "Token无效"
- 有效 → 生成新JWT + 新RefreshToken → 旧Token失效 → 返回新Token
```

#### 4.2.6 退出登录

```
POST /api/auth/logout

Request Header:
Authorization: Bearer {token}

Response:
{
  "code": 0,
  "message": "已退出登录"
}

业务逻辑：
- 从Redis删除当前RefreshToken
- 从Redis删除当前JWT黑名单（可选）
- 客户端清除Keychain/Keystore
```

#### 4.2.7 修改密码

```
PUT /api/auth/password

Request Header:
Authorization: Bearer {token}

Request:
{
  "oldPassword": "abc12345",
  "newPassword": "xyz67890"
}

Response:
{
  "code": 0,
  "message": "密码修改成功，请重新登录"
}

字段校验：
- oldPassword：必填，非空
- newPassword：必填，正则同注册，不能与旧密码相同（返回1015）

业务逻辑：
- 旧密码错误 → 返回1009
- 更新密码 → Redis清除该用户所有Token → 返回成功
```

#### 4.2.8 重置密码

```
POST /api/auth/password/reset

Request:
{
  "phone": "13800138000",
  "smsCode": "123456",
  "newPassword": "xyz67890"
}

Response:
{
  "code": 0,
  "message": "密码重置成功"
}

业务逻辑：
- 手机号未注册 → 返回1010
- 验证码校验同注册
- 更新密码 → Redis清除该用户所有Token
```

### 4.4 数据模型

#### 用户表（users）

```sql
CREATE TABLE users (
    id          BIGSERIAL PRIMARY KEY,
    user_id     VARCHAR(32)   NOT NULL UNIQUE,      -- 业务ID，如 u_a1b2c3
    phone       VARCHAR(20)   NOT NULL UNIQUE,      -- 手机号（AES加密存储）
    password    VARCHAR(255)  NOT NULL,              -- BCrypt加密
    nickname    VARCHAR(50)   DEFAULT '',            -- 昵称（默认手机号脱敏）
    avatar      VARCHAR(500)  DEFAULT '',            -- 头像URL
    email       VARCHAR(100)  DEFAULT '',            -- 邮箱
    is_verified BOOLEAN       DEFAULT FALSE,         -- 是否实名认证
    credit_score INTEGER      DEFAULT 80,           -- 信誉分
    credit_level VARCHAR(20)  DEFAULT '铜牌',        -- 信誉等级
    status      VARCHAR(20)   DEFAULT 'active',      -- active | locked | banned
    language    VARCHAR(10)   DEFAULT 'zh-CN',       -- 语言偏好
    created_at  TIMESTAMP     DEFAULT NOW(),
    updated_at  TIMESTAMP     DEFAULT NOW()
);

CREATE INDEX idx_users_phone ON users(phone);
CREATE INDEX idx_users_user_id ON users(user_id);
```

#### Token存储（Redis）

```
Key: jwt:{userId}          → JWT Token
Key: refresh:{userId}       → RefreshToken
Key: sms:{phone}:{type}    → 验证码
Key: login_attempt:{phone} → 登录失败次数
Key: user_lock:{phone}     → 账号锁定标记
```

## 五、安全规范

| 规范 | 说明 |
|------|------|
| 密码存储 | BCrypt加密，cost=12 |
| 密码传输 | HTTPS + 请求体，不做前端加密 |
| Token存储 | iOS Keychain / Android Keystore（非SharedPreferences） |
| JWT有效期 | 24小时 |
| RefreshToken有效期 | 30天 |
| 验证码有效期 | 5分钟 |
| 验证码重发间隔 | 60秒 |
| 手机号存储 | 数据库AES-256加密 |
| 登录失败限制 | 5次错误锁定30分钟 |
| 验证码日上限 | 10次/手机号/天 |
| SQL注入防护 | 使用JPA参数绑定，禁止拼接SQL |
| XSS防护 | 输入验证+输出编码 |
